  High Impact — Missing Analysis Capabilities

  Radiation-Cooled Extension Model
  Large nozzle extensions (like on the RL-10) are uncooled and reach equilibrium wall temperature via radiation. The current model assumes regenerative cooling everywhere. A separate equilibrium-radiation model for the extension section would be realistic.


  ---
  Medium Impact — Code Quality & Professionalism

  Serialization / Design Database
  There is no way to save a design to disk and reload it. Adding Jackson-based JSON serialization to NozzleDesignParameters, CharacteristicNet, and results would make the library usable as part of a persistent workflow.

  Proper Logging
  Main.java uses System.out throughout. ChemistryModel uses SLF4J but nothing else does. Consistent SLF4J usage across all classes would make the library production-grade.

  ---
  Medium Impact — Validation & Testing

  Integration Test: Full Design Flow
  No test exercises the full pipeline: NozzleDesignParameters → CharacteristicNet → NozzleContour → HeatTransferModel → NASASP8120Validator. A single end-to-end integration test would catch cross-component regressions.

  ---
  Lower Impact — Nice to Have

  - Unit conversion utilities — the library mixes SI throughout but users often work in imperial (lbf, inches, psia). A Units class would help at system boundaries.
  - CGNS binary export — currently just delegates to Plot3D. True CGNS would be compatible with more CFD solvers.
  - Dual-bell nozzle — altitude-compensating geometry that transitions between two contours; relevant for single-stage vehicles.
  - Coolant channel sizing — HeatTransferModel takes coolantHeatTransferCoeff as a given but doesn't compute it. A simple channel sizing sub-model (pressure drop, flow velocity, nucleate boiling margin) would close the loop.
